## 1. 시스템 요구사항

| 항목 | 요구사항 |
|------|----------|
| **운영체제** | Windows 10 / Windows 11 (64비트) |
| **하드웨어** | Euresys Grablink Full (PC1622) 프레임 그래버 |
| **카메라** | Crevis TC-A160K (Camera Link) |
| **드라이버** | Euresys MultiCam 드라이버 (설치 완료 상태) |
| **디스크** | 설치 공간 약 200MB + 이미지 저장 공간 별도 |

> `.NET` 런타임은 설치 파일에 포함되어 있어 별도 설치가 필요하지 않습니다.

## 2. 설치

### 2-1. 인스톨러 실행

배포된 인스톨러 파일을 더블클릭합니다.

```
PeanutVision Setup {버전}.exe
```

### 2-2. 설치 경로 선택

설치 마법사가 열리면 설치 경로를 선택합니다.

- 기본 경로: `C:\Program Files\PeanutVision`
- 경로를 변경하려면 **Browse** 버튼을 눌러 원하는 폴더를 선택합니다.

> **주의:** `C:\Program Files\` 내에 설치하는 것을 권장합니다. 앱 실행 파일과 리소스는 이 경로에 설치되고, 사용자 데이터(설정 파일, 이미지 DB)는 별도의 쓰기 가능한 경로에 자동 저장됩니다. ([사용자 데이터 위치](#7-사용자-데이터-위치) 참고)

### 2-3. 설치 완료

설치가 완료되면 자동으로 다음이 생성됩니다.

- **바탕화면 바로가기**: `PeanutVision`
- **시작 메뉴 항목**: `PeanutVision`

## 3. 앱 시작

### 3-1. 실행

바탕화면의 **PeanutVision** 바로가기를 더블클릭합니다.

### 3-2. 시작 순서

앱이 열릴 때 내부적으로 다음 순서로 동작합니다.

```
1. Electron 메인 프로세스 시작
      ↓
2. C# 백엔드(PeanutVision.Api.exe) 자동 실행
      ↓
3. 백엔드 준비 완료 대기 (최대 30초)
      ↓
4. 로딩 화면 → React UI 표시
```

로딩 화면이 표시되는 동안에는 정상적으로 초기화 중입니다. 보통 5초 이내에 UI가 표시됩니다.

### 3-3. 단일 인스턴스 제한

PeanutVision은 **한 번에 하나의 인스턴스만** 실행할 수 있습니다.

이미 실행 중인 상태에서 바로가기를 다시 클릭하면 오류 메시지가 표시되고 새 창이 열리지 않습니다. 기존 창을 사용하세요.

> 이 제한은 MultiCam 드라이버(하드웨어)를 동시에 두 프로세스가 접근하는 것을 방지하기 위한 것입니다.

---

## 4. 앱 종료

### 4-1. 종료 방법

창의 **X 버튼**을 클릭하거나 `Alt + F4`를 누릅니다.

### 4-2. 종료 순서

단순히 창이 닫히는 것이 아니라, 안전한 종료를 위해 다음 순서로 처리됩니다.

```
1. X 버튼 클릭
      ↓
2. Electron이 C# 백엔드에 종료 신호 전송 (POST /shutdown)
      ↓
3. C# 백엔드가 MultiCam 드라이버 리소스 해제 (McDelete, McCloseDriver)
      ↓
4. 백엔드 프로세스 종료 (최대 3초 대기)
      ↓
5. Electron 종료
```

> **강제 종료(작업 관리자 등)는 피하세요.** MultiCam 드라이버가 정상적으로 해제되지 않으면 다음 실행 시 하드웨어 초기화가 실패할 수 있습니다. 이 경우 PC를 재시작하면 해결됩니다.

## 5. 업그레이드

### 5-1. 방법

새 버전의 인스톨러를 실행합니다.

```
PeanutVision Setup {새버전}.exe
```

기존 버전을 먼저 삭제할 필요가 없습니다. 인스톨러가 자동으로 이전 버전을 교체합니다.

### 5-2. 사용자 데이터 유지

업그레이드 후에도 다음 데이터는 유지됩니다.

- 이미지 저장 경로 설정
- 촬영된 이미지 목록 (DB)
- 기타 사용자 설정

---

## 6. 삭제

### 6-1. 방법 A — 시작 메뉴

**시작 메뉴** → `PeanutVision` 우클릭 → **제거**

### 6-2. 방법 B — 제어판

**제어판** → **프로그램 및 기능** → `PeanutVision` 선택 → **제거**

### 6-3. 사용자 데이터 처리

앱을 제거해도 **사용자 데이터는 자동으로 삭제되지 않습니다.**

- 보존됨: `%APPDATA%\PeanutVision\` 폴더 (설정 파일, 이미지 DB)
- 삭제됨: 설치 경로의 앱 파일, 바탕화면/시작 메뉴 바로가기

데이터를 완전히 삭제하려면 제거 후 `%APPDATA%\PeanutVision\` 폴더를 수동으로 삭제하세요.

## 7. 사용자 데이터 위치

앱 실행 중 생성·수정되는 모든 데이터는 아래 경로에 저장됩니다.

```
C:\Users\{사용자명}\AppData\Roaming\PeanutVision\
```

| 파일 | 내용 |
|------|------|
| `peanut-vision.db` | 촬영된 이미지 목록 (SQLite) |
| `image-save-settings.json` | 이미지 저장 경로 설정 |

> `%APPDATA%`를 파일 탐색기 주소창에 입력하면 해당 폴더로 바로 이동할 수 있습니다.

이미지 파일 자체는 설정된 **Output Directory** 경로에 저장됩니다 (기본값: 앱 설치 경로 내 `CapturedImages/`).

## 8. 트러블슈팅

| 증상 | 원인 | 해결 방법 |
|------|------|-----------|
| 로딩 화면에서 30초 후 오류 다이얼로그 | 백엔드 시작 실패 (드라이버 문제 등) | MultiCam 드라이버 설치 여부 확인. 문제가 지속되면 PC 재시작 |
| "이미 실행 중" 오류 메시지 | 다른 PeanutVision 프로세스가 실행 중 | 작업 관리자에서 `PeanutVision.Api.exe` 종료 후 재시작 |
| 앱 종료 후에도 프로세스가 남아있음 | 강제 종료 또는 비정상 종료 발생 | 작업 관리자에서 `PeanutVision.exe`, `PeanutVision.Api.exe` 수동 종료 |
| 다음 실행 시 하드웨어 초기화 실패 | 이전 실행에서 MultiCam 드라이버가 정상 해제되지 않음 | PC 재시작 |
| 촬영 후 이미지 파일을 찾을 수 없음 | Output Directory 경로 설정 확인 필요 | 앱 설정 패널에서 **Image Save Settings** → **Output Directory** 경인 확인 |
